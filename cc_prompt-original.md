# QED プロジェクト：診断モード (SYN) 統合仕様書 (V1.8)

## 1. 外部ソースからのインポート
- 隣接するディレクトリ `C:/dev/subutomo-syn/src/SubutomoProtocol.js` を、本プロジェクトの `js/lib/` フォルダへコピーして統合してください。
- リザルト画面でスコアを表示する際、このモジュールの `encode()` を使用して「スコアコード（QED-XXXX...）」を表示する実装を追加してください。

## 2. 診断モード UI の新規実装
- `js/ui/MaintenanceUI.js` を新規作成し、ブラウザ上で動作するデバッグコンソールを実装してください。
- **視覚演出要件**:
  - 起動時に画面を3回白くフラッシュ（点滅）させ、全画面を黒背景のオーバーレイで覆うこと。
  - テキストは1文字ずつ表示されるタイプライター風の演出を適用すること。
  - フォントはレトロな等幅フォント、カラーはグリーン（#00FF00）系統を推奨。
- **入力要件**:
  - キーボード入力を取得し、画面上には `*` でマスク表示すること。
  - 小文字入力は内部的にすべて大文字に変換してから判定すること。

## 3. 認証ロジックと秘匿ルール（最重要）
以下の文字列を扱う際、ソースコード内にリテラル（生文字列）を一切残さないでください。

- **問いかけ（Prompts）**:
  - A: "YAMA"
  - B: "HADAKANOBUTAGAIRU"
  - ※ これらは Base64 または Hex 形式で難読化して保持し、表示時のみ復元すること。
- **回答（Answers）**:
  - 正解1: "KAWA"
  - 正解2: "FUJISAN"
  - 正解3: "SUBUTA"
  - ※ これらは SHA-256 等のハッシュ値のみを定数として保持し、入力値のハッシュと比較すること。

## 4. 階層構造（認証フロー）
- **Layer 1 & 2**: プロンプト (テキストA)? を表示。
  - "KAWA" 入力 → 一般保守員権限付与。
  - "FUJISAN" 入力 → 上級保守員権限付与。
  - **空エンター（Empty Enter）** 入力 → Layer 3 へ移行。
- **Layer 3 (隠し階層)**: プロンプト (テキストB)? を表示。
  - "SUBUTA" 入力 → スーパーバイザー権限付与。

## 5. 統合と終了
- メニュー画面の「その他（OTHER）」から、この MaintenanceUI を起動できるようにフックを追加してください。
- 終了時（または認証失敗時）は、スムーズにメニュー画面へ戻るようにしてください。
- 今後の報告、コードコメント、画面メッセージはすべて日本語で行ってください。
